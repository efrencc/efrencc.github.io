Resources and Resource Groups

What Are Resources?

Resources in Azure are any services you use or provisioned within the Azure platform. Anytime you create/provision a resource it will be placed with the desired Resource Group that serves as a logical organization management. Remember the overall Azure organization:

Resources can be of different types, the main ones being compute, storage, databases, and AI services. Here are a few examples of resources within Azure according to type:

  1. Compute
    • VMs (Virtual Machines)
    • Containers
    • Serverless Functions
  2. Storage
    • Blob Storage
    • Azure Files
    • Azure Data Lakes
  3. Databases
    • Azure SQL DB
    • Cosmos DB
  4. AI and Machine Learning
    • Azure Cognitive Services
    • Azure OpenAI

We will now show you how to create a resource group step by step. The specific process may change if Azure's interface gets updated, but the principle is the same.

Creating an Azure Resource Group

  1. Open the “Resource Groups” Services
  2. Once on the Resource group page click “+Create”
  3. Enter project details. It will ask you for subscription, resource group name, and region. Below is a description of these:
    1. Subscription: The subscription in which you want to build the resource group. This sets which subscription gets billed.
    2. Resource Group: Create a name for this resource group.
    3. Region: Select a region where you want this resource group to be placed. (Not all regions have access to all services.)
  4. (Optional) add Tag/s to help with Organization. A tag is useful for organization as you can group resources with Tags.
  5. Review your resource group creation and Press “Create”.

Now anytime you want to create / Request a service you can select a specific resource group to help manage.

Roles

In Azure, roles define permissions for accessing and managing Azure resources. Roles are crucial for maintaining security and governance within an Azure environment by controlling who can perform specific actions on resources. Azure provides several built-in roles with predefined sets of permissions, such as Owner, Contributor, and Reader, covering a wide range of responsibilities from full control to read-only access. Additionally, custom roles can be created to tailor permissions according to specific organizational requirements.
