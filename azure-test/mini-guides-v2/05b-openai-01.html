Azure OpenAI - Models, Regions and Quotas

Privacy and Security

The GPT models available through Azure OpenAI are not hosted by OpenAI itself. That is, the information you send to Azure is not sent, processed, or collected by OpenAI (this is true as of April, 2024, it may change in the future).

Regions

You can choose the server region where you want to create your resource (for example, East US, West US, etc.). While ideally you would choose the closest region to your location, not all services are available in all regions. To see which models are available in which regions, follow this link.

Model Deployments

When beginning to develop your own AI services within Azure AI using its AI services, you'll require a baseline model for deployment or fine-tuning. Azure offers a wide range of models to select from, whether you're initiating a chatbot with GPT, generating images from text using DALL-E, or converting speech to text or vice versa using Whisper. Azure Services provide various variants of each model, allowing for flexibility in choosing the most suitable option for your project.

Quotas

For each model, there is a limit associated to it per region and per subscription. What this limit, or quota, means, depends on the model. For example, if you use GPT 3.5 turbo, the limit is in terms of “tokens per minute”, while if you select a DALL-E the limit is in terms of the number of concurrent requests. When you create a resource using a model, you select how much of the quota is assigned to that resource. You can assign all your quota of divide it into different resources/deployments. For example say I make a resource for GPT 3.5 Turbo in the East US region. My limit is 240K tokens per minute, and I assign 120K to this resource. If I create another resource in the same region and using the same model, I can only use 120K max. I could alternatively assign 60K and leave 60K to space for a possible third resource.
